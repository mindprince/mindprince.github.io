Setup ganglia monitoring on your Amazon EC2 clusters

Nov 23rd, 2013

I started four instances running Amazon Linux AMI 2013.09.1 - ami-4296c310 (64-bit) in the Singapore region ap-southeast-1. The security group for these instances allowed inbound SSH and HTTP from my IP address and all traffic from other instances in the same security group.

ON THE MASTER INSTANCE

Installation

Install ganglia-gmond, ganglia-gmetad and ganglia-web.

sudo yum install ganglia-gmond ganglia-gmetad ganglia-web

Edit ganglia configuration files

sudo vim /etc/ganglia/gmond.conf 

sudo vim /etc/ganglia/gmetad.conf

Edit apache httpd configuration files

sudo vim /etc/httpd/conf/httpd.conf  

sudo vim /etc/httpd/conf.d/ganglia.conf

Start daemons

sudo service gmetad start
sudo service gmond start  
sudo service httpd start

ON THE SLAVE INSTANCES

Installation

sudo yum install ganglia-gmond

Edit ganglia configuration files

sudo vim /etc/ganglia/gmond.conf

Start daemons

sudo service gmond start
